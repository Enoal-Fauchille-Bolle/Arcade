Arcade

Description

Arcade is a modular retro gaming platform developed in C++, allowing users to choose and play different games while keeping track of player scores. The project uses a system of dynamic libraries to load and execute games and graphical interfaces on the fly.

Features

  • Interactive game selection menu
  • Support for multiple interchangeable display libraries at runtime
  • Player score management
  • Dynamic loading of games and graphical interfaces as plugins
  • Modular architecture enabling easy addition of new games or interfaces

Compilation and Installation

Dependencies

Make sure to install the required libraries before compiling the project:

  • C++17 or later
  • Makefile or CMake
  • libdl for dynamic library handling
  • SDL2, nCurses (and potentially other display libraries of your choice)

Compilation with Makefile

make

Compilation with CMake

mkdir build && cd build
cmake .. -G "Unix Makefiles" -DCMAKE_BUILD_TYPE=Release
cmake --build .

Once the compilation is complete, the arcade binary will be generated at the project root.

Usage

The program must be launched with a display library as an argument:

./arcade ./lib/arcade_ncurses.so

Main Controls

  • Switch display library
  • Switch game
  • Restart the game
  • Return to the main menu
  • Exit the program

Project Structure

/arcade
│── src/ # Main source code
│── include/ # Header files
│── lib/ # Dynamic libraries (games & graphical interfaces)
│── games/ # Game implementations
│── graphics/ # Graphical library implementations
│── doc/ # Documentation
│── README.md # This file

Adding a Game or a Graphical Library

Adding a Game

Games must be developed as dynamic libraries (.so) and placed in the ./lib/ folder. They must follow a defined interface to be compatible with the main program.

Adding a Graphical Library

Like games, display libraries must be dynamic and follow a standard interface to be recognized and loaded by the program.
